# الدليل المعماري لربط النظام بالكامل (Architecture Guide)

لتحويل هذا الكود إلى نظام حقيقي متكامل يعمل بنسبة 100% باستخدام أدوات **Google Cloud**، ستحتاج إلى المكونات التالية وربطها ببعضها:

### 1. قاعدة البيانات (Database)
*   **الأداة:** **Google Cloud SQL (PostgreSQL)**
*   **الدور:** تخزين بيانات العملاء (Customers)، المحادثات (Conversations)، الرسائل (Messages)، والقوالب (Templates).
*   **الذكاء الاصطناعي:** يجب تفعيل إضافة `pgvector` داخل PostgreSQL لتخزين الذاكرة (Embeddings) الخاصة بالمحادثات السابقة للعملاء لكي يتذكرها الذكاء الاصطناعي.

### 2. الواجهة الأمامية (Frontend - Dashboard)
*   **الأداة:** **Firebase Hosting** أو **Google Cloud Run**
*   **الدور:** استضافة الكود الذي تم إنشاؤه في هذا المشروع (React SPA). هذه الواجهة سيستخدمها فريق المبيعات لمتابعة التحليلات، التدخل البشري في المحادثات، وتحليل السير الذاتية.

### 3. الواجهة الخلفية (Backend & Webhooks)
*   **الأداة:** **Google Cloud Run** (باستخدام Node.js/Express) أو **n8n** (مستضاف على Cloud Run).
*   **الدور:** 
    *   استقبال الـ Webhooks من سيرفرات Meta (واتساب وانستجرام) في أجزاء من الثانية.
    *   حفظ الرسائل في قاعدة البيانات.
    *   إرسال الرسالة إلى Gemini API لتوليد الرد.
    *   إرسال الرد مرة أخرى إلى العميل عبر Meta API.

### 4. الذكاء الاصطناعي (AI Engine)
*   **الأداة:** **Vertex AI (Gemini 2.5 Flash)**
*   **الدور:** 
    *   تحليل السير الذاتية (CV Analyzer) واستخراج نقاط القوة والضعف.
    *   العمل كمندوب مبيعات (Sales Agent) للرد على استفسارات العملاء على الواتساب والانستجرام بناءً على سياق المحادثة.

### 5. ربط واتساب وانستجرام (WhatsApp & Instagram API)
*   **الأداة:** **Meta Developer Portal**
*   **الخطوات:**
    1. إنشاء تطبيق (App) في Meta.
    2. إضافة منتج WhatsApp و Instagram Graph API.
    3. إعداد الـ **Webhooks** لكي تشير إلى رابط الـ Backend الخاص بك (مثال: `https://your-backend-url.a.run.app/webhook/whatsapp`).
    4. الحصول على `Access Token` دائم لاستخدامه في الـ Backend لإرسال الرسائل.

## 🚀 خطوات الربط الفعلي (How to connect everything)

لكي تجعل كل شيء "مربوطاً ببعضه وجاهزاً"، اتبع هذا التسلسل:

1.  **تجهيز البنية التحتية (Infrastructure):**
    *   افتح Google Cloud Console.
    *   أنشئ قاعدة بيانات PostgreSQL (Cloud SQL).
    *   أنشئ حساب Service Account بصلاحيات Vertex AI و Cloud SQL.
2.  **برمجة الواجهة الخلفية (The Middleware/Backend):**
    *   تحتاج إلى كود Node.js بسيط يعمل كـ "وسيط".
    *   **المسار الأول:** يستقبل رسائل الواتساب -> يحفظها في الـ DB -> يرسلها لـ Gemini -> يأخذ الرد -> يرسله للواتساب -> يحفظ الرد في الـ DB.
    *   **المسار الثاني:** يوفر API للواجهة الأمامية (Dashboard) لجلب الإحصائيات والمحادثات من قاعدة البيانات.
3.  **نشر الواجهة الخلفية (Deploy Backend):**
    *   ارفع كود الـ Node.js على **Google Cloud Run** ليعمل على مدار الساعة ويتحمل الضغط.
4.  **ربط Meta بالواجهة الخلفية:**
    *   خذ الرابط الذي أعطاك إياه Cloud Run، وضعه في إعدادات Webhook في منصة Meta Developers.
5.  **نشر الواجهة الأمامية (Deploy Frontend):**
    *   خذ كود الـ React (الموجود في هذا المشروع)، وقم بتعديل ملفات الـ Services لكي تتصل بالـ API الخاص بالواجهة الخلفية بدلاً من البيانات الوهمية (Mock Data).
    *   ارفعه على **Firebase Hosting**.

بمجرد إتمام هذه الخطوات الـ 5، سيكون لديك نظام حقيقي متكامل يعمل 100% مثل الشركات الكبرى!
